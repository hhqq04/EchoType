一个轻量级的 Windows 自动键盘输入模拟器，基于 C 语言和 Win32 API 编写。支持中文输入，通过模拟底层键盘事件，将粘贴的文本自动输入到任何目标窗口中。

✨ 功能特点
Unicode 支持：完美支持中文、日文、特殊符号输入。
触发机制：录入文本后，将光标移至目标窗口，**按下回车键 (Enter)** 即可触发输入，无需倒计时。
无需安装：编译后的 `.exe` 文件可直接运行，绿色免安装。
多行支持：自动识别换行符并模拟回车键。

 🚀 快速开始

1. 下载或编译
你可以直接下载 Release 中的 `exe` 文件，或者按照下方教程自行编译。

 2. 使用步骤
1. 打开程序。
2. 粘贴或输入你需要模拟输入的文本内容。
3. 按 `Ctrl + Z`，然后按 `Enter` 确认输入结束。
4. **关键步骤**：将鼠标移动并点击你想要输入的应用程序（如记事本、IDE、游戏输入框）。
5. 再次按下 **回车键 (Enter)**。
6. 程序将自动模拟键盘操作，将文字“打”出来。

🛠️ 编译指南 (Compilation)

本项目使用 Windows API，建议使用 GCC (MinGW) 进行编译。

⚠️ 重要提示**：为了避免在其他电脑上运行时出现 `libgcc_s_dw2-1.dll` 或 `libwinpthread-1.dll` 缺失的错误，请务必使用 `-static` 参数进行静态链接。

```bash
gcc main.c -o AutoTyper.exe -static
```
 ⚙️ 调整速度

如果你想修改打字速度，请修改源码中的 `Sleep` 参数：

打字间隔：修改 `sendUnicodeChar` 循环中的 `Sleep(2);`（单位：毫秒）。数值越大，打字越慢。
启动延迟：修改检测到回车键后的 `Sleep(200);`。

 📝 代码说明

   `SendInput`：核心 API，用于模拟键盘硬件事件。
  `MultiByteToWideChar`：将 GBK 编码（Windows 控制台默认）转换为 Unicode，确保中文正常显示。
   `GetAsyncKeyState`：异步检测按键状态，用于实现回车键触发功能。
